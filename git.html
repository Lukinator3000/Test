Git & GitHub Tips

Git is a open source version control system utilized to track file version changes over time to facilitate code testing and collaboration.


GitHub is a cloud-based hosting service for Git repositories that lets you work on project from different devices.


Using the two together, you can easily make, test, save, and manage files in a local repository and send them to a remote repository online for other devices to access.


Important Git Commands

git init

- Initializes the current folder as a project for git to track


git add .

- Stages new or modified files in local repository, allowing those files to be committed (if you want to stage a specific file, replace . with the file name)


git status

- Checks to see what files are staged if any


git commit -m "[message]"

- Commits (saves) staged files, making them able to be pushed to a GitHub repository (also with a commit message in quotation marks)


git remote add origin https://github.com/[username]/[repository-name]

- Connects the local repository with the remote GitHub repository using a GitHub link (in the form of the url above)


git branch -M main

- Changes main branch name to "main"


git push -u commit [branch-name]

- Pushes committed files of current branch to the connected GitHub repository


git checkout -b [new-branch-name]

- Creates a new branch in the repository with the name replacing [new-branch-name] and goes to that branch


git checkout [branch-name]

- Switches branch to the one named in [branch-name]


git merge [branch-name]

- Combines additional branch and its file changes with main branch
